# 初心者向け AWS Lambda セットアップガイド

非エンジニアの方向けに、1ステップずつ画面を見ながら設定できるガイドです。

---

## 準備物

- **AWS アカウント**（なければ作成）
- **Chatwork API トークン**（会社のシステム管理者に確認）
- **Mac のターミナル**（コマンド実行用）

---

## 全体の流れ

1. **ローカルでコード準備**（Mac で実行）
2. **AWS Lambda 関数を作成**（AWS コンソール操作）
3. **コードをアップロード**（AWS コンソール操作）
4. **環境変数を設定**（AWS コンソール操作）
5. **テスト実行**（AWS コンソール操作）
6. **毎日9時に自動実行を設定**（AWS コンソール操作）

---

---

## ステップ 1: ローカル準備（Mac でのコマンド実行）

このステップは Mac のターミナルで実行します。

### 1.1 ターミナルを開く

**操作：**
1. Mac で **Spotlight 検索** を開く（`⌘ + Space`）
2. 「ターミナル」と入力
3. 「ターミナル.app」をクリック

**画面イメージ：**
```
ターミナルが起動すると、黒い画面が表示されます
```

### 1.2 デプロイメント ZIP ファイルを作成

**ターミナルに以下をコピペして実行：**

```bash
cd ~/AIproduct/Claude/ga4-kpi-batch
pip install -r requirements.txt -t python/
```

**待ってください：** 30秒～1分程度かかります。完了するまで待ちます。

次に、ZIP ファイルを作成：

```bash
zip -r lambda_deployment.zip \
    lambda_handler.py \
    main.py \
    ga4_client.py \
    gsc_client.py \
    chatwork_client.py \
    report_builder.py \
    sheets_client.py \
    python/
```

**確認：** ファイルが作成されたか確認

```bash
ls -lh lambda_deployment.zip
```

以下のような出力が出れば成功：
```
-rw-r--r--  1 kazuya  staff   48M  3月  1 12:34 lambda_deployment.zip
```

## ステップ 3: Lambda 関数を作成

### 3.1 Lambda のページに移動

**操作：**
1. コンソールの上の検索欄に「Lambda」と入力
2. 「Lambda」をクリック

**画面イメージ：**
```
[AWS Management Console]
  検索欄: Lambda ← ここに入力
```

### 3.2 関数を作成ボタンをクリック

**操作：**
1. 画面右側の **「関数を作成」** ボタンをクリック

**画面イメージ：**
```
[Lambda コンソール]
  右上：「関数を作成」ボタン ← クリック
```

### 3.3 基本設定を入力

**操作：** 以下の項目を設定

| 項目 | 入力値 |
|------|--------|
| **関数名** | `ga4-kpi-batch` |
| **ランタイム** | `Python 3.12` |
| **実行ロール** | 「Lambda 用に新しいロールを作成」を選択 |

**画面イメージ：**
```
[関数を作成]
  関数名: ga4-kpi-batch
  ランタイム: Python 3.12 ← ドロップダウンで選択
  実行ロール:
    ◉ Lambda 用に新しいロールを作成 ← 選択
    ○ 既存のロールを使用
```

### 3.4 関数を作成をクリック

**操作：**
画面下の **「関数を作成」** ボタンをクリック

**待ってください：** 5～10秒で関数が作成されます。

**成功画面：**
```
[ga4-kpi-batch 関数が作成されました]
  関数の詳細ページが表示される
```

## ステップ 5: 環境変数を設定

環境変数とは、プログラムに必要な「設定値」のようなものです。

### 5.1 設定ページを開く

**操作：**
1. Lambda 関数の詳細ページで、上部の **「設定」** タブをクリック

**画面イメージ：**
```
[Lambda 関数]
  タブ: [コード]  [設定] ← クリック
```

### 5.2 環境変数を編集

**操作：**
1. 左側メニューで **「環境変数」** をクリック
2. **「編集」** ボタンをクリック

**画面イメージ：**
```
[設定]
  左メニュー:
    - 一般設定
    - 環境変数 ← クリック

  [編集] ボタン ← クリック
```

### 5.3 Chatwork API トークンを追加（重要：最初に入力）

**操作：**
1. **「環境変数の追加」** ボタンをクリック
2. 以下を入力：

| キー | 値 |
|------|-----|
| **キー** | `CHATWORK_API_TOKEN` |
| **値** | あなたの Chatwork API トークン |

**キーの入力例：**
```
[キー]        [値]
CHATWORK_API_TOKEN    ← ここに入力
```

**値の入力方法：**
- Chatwork API トークンは会社の管理者に確認してください
- 長い英数字の文字列です
- コピーして貼り付けてください

### 5.4 その他の環境変数を追加

**操作：** 以下の表から 1つずつ「環境変数の追加」をクリックして入力

| キー | 値 |
|------|-----|
| `CHATWORK_ROOM_ID` | `421983199` |
| `GA4_PROPERTY_ID` | `374086323` |
| `SC_SITE_URLS` | `https://blockchaingame.jp/,https://news.blockchaingame.jp/` |
| `OUTPUT_SHEET_ID` | `12W1b-BDg-smRWYqKmGqdmoKOswagbDQrlY7AKJ0BG-Q` |
| `TZ` | `Asia/Tokyo` |
| `MAX_CHATWORK_CHARS` | `6000` |
| `MIN_GSC_IMP` | `30` |
| `MIN_GSC_CLICKS_SUM` | `2` |
| `MIN_GSC_CLICK_DIFF` | `1` |
| `MIN_GA4_SESSIONS` | `5` |
| `MIN_GA4_7D_AVG` | `15` |
| `DROP_RATIO` | `0.6` |
| `MIN_GA4_ABS_DROP` | `5` |
| `CTR_OPP_MIN_IMP` | `30` |
| `CTR_OPP_POS_MIN` | `2` |
| `CTR_OPP_POS_MAX` | `10` |

**入力例：**
```
[キー]            [値]
CHATWORK_ROOM_ID  421983199
GA4_PROPERTY_ID   374086323
...
```

### 5.5 GOOGLE_CREDENTIALS_JSON を追加（最後に）

このステップが少し複雑です。丁寧に進めます。

**操作：**
1. **「環境変数の追加」** をクリック
2. **キー** に以下を入力：
   ```
   GOOGLE_CREDENTIALS_JSON
   ```

3. **値** にコピーする内容を取得（ターミナルで実行）：

**ターミナルで実行：**
```bash
cat ~/AIproduct/Claude/ga4-kpi-batch/sa-key.json | jq -c '.'
```

このコマンドの出力（長い1行）をコピーします。

**コピーの方法：**
1. ターミナルの出力を選択
2. `⌘ + C` でコピー
3. AWS コンソールの **値** フィールドに貼り付け（`⌘ + V`）

**画面イメージ：**
```
[ターミナル]
$ cat ~/AIproduct/Claude/ga4-kpi-batch/sa-key.json | jq -c '.'
{"type":"service_account","project_id":"blockchaingameinfo-487515",...}
                                                                     ↑
                                    この長い文字列をすべて選択＆コピー
```

### 5.6 保存

**操作：**
画面下の **「保存」** ボタンをクリック

**待ってください：** 設定が保存されます（5～10秒）

**成功メッセージ：**
```
✓ 環境変数が正常に更新されました
```

## ステップ 8: 毎日9時に自動実行を設定

### 8.1 EventBridge コンソールを開く

**操作：**
1. AWS コンソール上部の検索欄に「EventBridge」と入力
2. 「EventBridge」をクリック

**画面イメージ：**
```
[AWS Management Console]
  検索欄: EventBridge ← ここに入力
```

### 8.2 ルールを作成

**操作：**
1. 左側メニューで **「ルール」** をクリック
2. **「ルールを作成」** ボタンをクリック

**画面イメージ：**
```
[EventBridge]
  左メニュー:
    - ダッシュボード
    - ルール ← クリック

  [ルールを作成] ボタン
```

### 8.3 ルール設定を入力

**操作：** 以下を入力

| 項目 | 入力値 |
|------|--------|
| **ルール名** | `ga4-kpi-batch-daily-9am` |
| **説明** | `毎日9時に GA4 日次レポートを実行` |
| **イベントバス** | `default` を選択 |

**画面イメージ：**
```
[ルールを作成]
  ルール名: ga4-kpi-batch-daily-9am
  説明: 毎日9時に GA4 日次レポートを実行
  イベントバス: default
```

### 8.4 スケジュール式を設定

**操作：**
1. **「ルールタイプ」** で **「スケジュール式」** を選択
2. **「スケジュール式」** というテキストボックスに以下を入力：
   ```
   cron(0 0 ? * * *)
   ```

**意味：** 毎日午前9時（日本時間）に実行

**画面イメージ：**
```
[スケジュール]
  ◉ スケジュール式
  ○ 定期レート

  スケジュール式:
  [cron(0 0 ? * * *)]
```

### 8.5 ターゲットを設定

**操作：**
1. 画面下の **「ターゲット」** セクションで **「AWS サービス」** を選択
2. ドロップダウンから **「Lambda 関数」** を選択
3. **「関数」** という欄で **`ga4-kpi-batch`** を選択

**画面イメージ：**
```
[ターゲット]
  AWS サービス ← 選択

  サービス: Lambda 関数 ← 選択
  関数: ga4-kpi-batch ← 選択
```

### 8.6 ルールを作成

**操作：**
画面下の **「ルールを作成」** ボタンをクリック

**成功メッセージ：**
```
✓ ルール ga4-kpi-batch-daily-9am が作成されました
```

## トラブルシューティング

### Q. テスト実行でエラーが出た

**A. エラーメッセージを確認：**

1. Lambda コンソールの **「テスト」** タブを見る
2. **「ログ」** セクションにエラーメッセージが表示されている
3. よくあるエラー：
   - **「環境変数が設定されていません」** → 環境変数を確認（ステップ5）
   - **「認証に失敗」** → `GOOGLE_CREDENTIALS_JSON` を確認
   - **「Chatwork に送信できない」** → トークンとルーム ID を確認

### Q. Chatwork に投稿されない

**A. 以下を確認：**

1. `CHATWORK_API_TOKEN` が正しく設定されているか
2. `CHATWORK_ROOM_ID` が `421983199` になっているか
3. テスト実行のログを確認（ステップ7）

### Q. 毎日9時に実行されない

**A. EventBridge の設定を確認：**

1. EventBridge コンソール → ルール
2. ルール `ga4-kpi-batch-daily-9am` が **有効** になっているか確認
3. ターゲットが `ga4-kpi-batch` Lambda 関数に設定されているか確認

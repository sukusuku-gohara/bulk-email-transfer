## 1. 背景・目的

### 1.1 プロジェクトの背景

meetsは、マッチングアプリと結婚相談所の中間層（Missing Middle）をターゲットとしたエージェント型マッチングプラットフォームである。

**課題認識**:
- マッチングアプリは「軽すぎる」（成婚率が低い、サポート不足）
- 結婚相談所は「重すぎる」（高額、敷居が高い）
- 既存のIBJ連盟は手数料が高額で、中小・個人相談所にとって負担が大きい

**市場機会**:
- 非IBJ連盟所属の中小・個人相談所（TMS、BIU、JBA等）
- 副業エージェント候補（YouTuber、Instagrammer、HR/Sales職等）
- Missing Middle層の婚活ニーズ（真剣だが高額相談所は避けたい層）

### 1.2 解決したい課題

| ユーザー種別 | 課題 | meetsの解決方法 |
|------------|------|----------------|
| **会員ユーザー** | マッチングアプリでは成婚に至らない、結婚相談所は高額 | 低価格（月額980円）でエージェントのサポートを受けながら婚活できるプラットフォームを提供 |
| **エージェント（中小相談所）** | IBJ連盟の高額会費、会員データベースの不足 | 低コストで会員データベースにアクセス、自由な料金設定、meetsが決済・運用を代行 |
| **副業エージェント候補** | 婚活業界への参入障壁が高い、システム構築コストが高い | meetsアカデミーで育成、プラットフォーム提供、CRM・決済システムを完備 |
| **アンバサダー** | 紹介ビジネスの収益化が困難 | 成婚料の一部をキックバック、紹介実績ダッシュボード提供 |

### 1.3 システムの目的

meetsは、以下の3つの成長柱で展開する。

1. **Agency（仲人×オープン化）**: 非IBJ連盟エージェントのネットワーク化、オープンプラットフォーム構築
2. **AI（仲人×AI）**: 将来的なAI導入によるマッチング精度向上、AIコーチング
3. **Global（仲人×グローバル化）**: 韓国・台湾・ベトナム等への海外展開

**Mission**: 「婚活の民主化」と「成婚の産業化」

## 2. 対象ユーザー / 課題

### 2.1 対象ユーザー

1. **会員ユーザー**: 結婚を真剣に考える18歳以上の独身者
2. **エージェント**: 非IBJ連盟所属の結婚相談所、副業仲人（YouTuber等）
3. **アンバサダー**: 会員紹介を行い成婚料の一部を受け取るパートナー
4. **プラットフォーム管理者**: meets運営チーム

### 2.2 ユーザーごとの課題と解決方法

#### 2.2.1 会員ユーザー（ペルソナ: 田中美咲、28歳、都内勤務、年収450万円）

**課題**:
- マッチングアプリで疲弊し、結婚相談所は高額で躊躇
- エージェントのサポートを受けながら、自分のペースで婚活を進めたい

**解決方法**:
- 月額980円の低価格でエージェントサポートを受けられる
- eKYC（本人確認）による安心・安全なプラットフォーム
- オンラインお見合い・匿名チャット機能で効率的に活動できる

#### 2.2.2 エージェント（ペルソナ: 山田花子、35歳、非IBJ連盟の個人相談所経営）

**課題**:
- 会員20名を抱えているが、IBJ連盟の高額な会費がネックで未加入
- 会員データベースが不足し、マッチング機会が限られている

**解決方法**:
- meetsで低コストに会員データベースを拡充
- 成婚料を自由に設定可能、meets経由決済で10%手数料のみ
- CRMでパイプライン管理、業務効率化

#### 2.2.3 アンバサダー（ペルソナ: 鈴木太郎、40歳、国際結婚相談所経営、YouTube登録者5万人）

**課題**:
- 自分の顧客（主に男性）をmeetsに紹介し、成婚料の一部をキックバックとして受け取りたい

**解決方法**:
- 専用紹介コード発行、紹介会員の成婚時に成婚料の一部（例: 30%）をキックバック
- 紹介実績ダッシュボードでリアルタイム表示

### 6.3 運用フロー

#### 6.3.1 会員登録フロー

1. 会員が段階的プロフィール入力（7画面、自動保存）

#### 6.3.2 お見合いフロー

1. エージェント（or 未所属会員本人）が相手に申し込み
2. 相手エージェント（or 相手本人）が承認/拒否
3. 承認 → 日程調整（カレンダーから候補日時3つ提示 → 相手が1つ選択）
4. 確定 → Zoom URL自動生成＋双方へLINE通知
5. リマインド通知（前日19時、当日3時間前、1時間前）
6. お見合い実施
7. フィードバック入力（また会いたい / NG / 本交際申込み）
8. 双方「また会いたい」 → 仮交際、匿名チャット開放

#### 6.3.3 成婚フロー

1. 本交際中の会員が「成婚退会」選択
2. エージェントが承認
3. 成婚料決済リンク即時発行（メール送信）
4. 会員が成婚料決済
5. meets側が10%手数料差し引き、エージェント・アンバサダーに配分
6. アカウント「成婚退会」ステータスに変更、ログイン不可

## 11. 付録

### 11.1 用語集

| 用語 | 定義 | 備考 |
|------|------|------|
| **Missing Middle** | マッチングアプリと結婚相談所の中間層を指す。「アプリは軽すぎるが、相談所は重すぎる層」 | meetsのターゲット層 |
| **エージェント** | meetsプラットフォーム上で会員をサポートする仲人。非IBJ連盟の中小・個人相談所、副業仲人（YouTuber等） | - |
| **アンバサダー** | 会員紹介を行い成婚料の一部をキックバックとして受け取るパートナー | - |
| **eKYC** | electronic Know Your Customer。運転免許証・マイナンバーカード等をOCR読取＋顔写真照合により自動照合する本人確認手法 | - |
| **お見合い** | 初回対面（オンラインまたはオフライン）。お見合い後のフィードバック結果により、仮交際へ進むか判断する | - |
| **仮交際** | お見合い後、双方が「また会いたい」を選択した状態。複数名と同時進行可能 | - |
| **本交際** | 仮交際から1対1専念に移行した状態。双方の「本交際申込み」＋エージェント承認が必要 | - |
| **成婚** | 本交際中の会員が「成婚退会」を選択し、エージェントが承認した状態。成婚料決済フローが開始される | - |
| **匿名チャット** | お見合い承認後、双方が個人LINE交換を不要にするために利用するチャット機能。個人情報の送信をブロックする | - |
| **中抜き** | プラットフォーム外で直接契約・金銭授受を行う違反行為。利用規約で禁止、違反時は違約金請求の可能性あり | - |
| **レベニューシェア（RS）** | 成婚料をmeets・エージェント・アンバサダーで配分する仕組み。meets側は各者から10%を手数料として徴収 | - |
| **パイプライン管理（CRM）** | エージェントが担当会員の活動フェーズ（登録→お見合い申込→お見合い実施→仮交際→本交際→成婚）を一覧表示する機能 | - |
| **NG判定** | お見合い後のフィードバックで「NG」を選択すること。双方向非表示となり、以降の検索結果から永久除外される（CHG-20260223-003） | - |
| **月間申請回数制限** | エージェント未所属ユーザーのお見合い申請回数を月間10回に制限する仕組み。上限到達時にエージェント所属への誘導を実施（CHG-20260226-001） | - |
| **システム利用料** | 仲人（エージェント）がmeetsに支払う利用料。担当会員1人あたり980円/月。仲人付き会員の月額会費に代わるもの（CHG-20260309-003） | - |
| **証明書有効期限** | 証明書類の受付可能な発行日からの期間。独身証明書・住民票は3ヶ月、年収証明書は1年（CHG-20260309-002） | - |

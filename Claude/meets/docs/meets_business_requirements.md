## 1. 概要（Executive Summary）

### 1.1 プロジェクト目的

meetsは、マッチングアプリと結婚相談所の中間層（Missing Middle）をターゲットとした**エージェント型マッチングプラットフォーム**である。「婚活の民主化」と「成婚の産業化」をMissionとし、以下3つの成長柱で展開する。

- **Agency**: 非IBJ連盟の中小・個人相談所および副業エージェントをネットワーク化
- **AI**: 将来的なAIエージェント導入によるマッチング精度向上
- **Global**: 韓国・台湾・ベトナム等への海外展開

### 1.2 対象ユーザー

1. **会員ユーザー**: 結婚を真剣に考える18歳以上の独身者
2. **エージェント**: 非IBJ連盟所属の結婚相談所、副業仲人（YouTuber等）
3. **アンバサダー**: 会員紹介を行い成婚料の一部を受け取るパートナー
4. **プラットフォーム管理者**: meets運営チーム

### 1.3 主要な事業成果（Business Outcome）

| 指標 | Year 1 目標 |
|------|-------------|
| エージェント登録数 | 1,000名 |
| 会員登録数 | エージェント × 平均50名 = 50,000名想定 |
| 成婚数 | 計測開始（目標値は運用後に設定） |
| 月次経常収益（MRR） | ~~会員月額980円 x アクティブ会員数~~（変更前） **【変更後】未所属会員980円 x 未所属アクティブ会員数 + エージェントシステム利用料980円 x 仲人付き会員数 + エージェント売上の10%（CHG-20260309-003）** |

> **[2026-03-09 変更 CHG-20260309-003]** MRR計算式を更新。仲人付き会員の課金はエージェントからのシステム利用料として徴収する形に変更。

### 1.4 開発スケジュール

- **2月**: 要件定義

- **4月**: 実装・テスト完了

### 3.1 MVP優先度の判断基準

以下の3条件を満たす要件をMVPに含める：

1. **ユーザー価値**: 会員が「お見合い→交際→成婚」の最小サイクルを完結できるか
2. **事業制約**: 2026年4月末までに実装可能か（開発期間3ヶ月、予算5,000万円）
3. **依存関係**: 他の要件の前提条件となっているか

## 4. ユーザーストーリー

### 4.1 会員ユーザー

**ペルソナ**: 田中美咲（28歳、都内勤務、年収450万円）
マッチングアプリで疲弊し、結婚相談所は高額で躊躇。エージェントのサポートを受けながら、自分のペースで婚活を進めたい。

#### US-001: 初回登録からお見合い申し込みまで
**As a** 会員候補者  
**I want to** プロフィールを段階的に入力し、eKYCで本人確認を完了させたい  
**So that** エージェントに自分を紹介してもらい、お見合いを始められる

**受け入れ基準**:
- 登録フォームが5〜7画面に分割されており、各画面で自動保存される
- eKYC提出後24時間以内に承認/差し戻しの通知が届く
- エージェントから連絡があり、希望条件のヒアリングが行われる
- 初回お見合い申し込み時に月額980円の決済画面が表示される
- エージェント未所属の場合、当月の残り申請回数（月間10回まで）がマイページに表示される

#### US-002: オンラインお見合いの実施
**As a** 会員ユーザー  
**I want to** オンラインでお見合いを実施し、相手の雰囲気を確認したい  
**So that** 仮交際に進むかどうかを判断できる

**受け入れ基準**:
- お見合い前日19時、当日3時間前、1時間前にLINE通知が届く
- システムが発行したZoom URLでビデオ通話ができる
- お見合い終了後、必須フィードバック（また会いたい/NG/本交際申込み＋コメント）を入力する画面が表示される
- 双方が「また会いたい」を選択した場合、匿名チャットが開放される

#### US-003: 匿名チャットでの安全なやり取り
**As a** 仮交際中の会員  
**I want to** 匿名チャットで相手とやり取りしたい  
**So that** 個人情報を交換せずに、安全に関係を深められる

**受け入れ基準**:
- チャット内でURL、電話番号、メールアドレス、LINE IDを送信しようとするとブロックされる
- 違反試行時に警告メッセージが表示される
- 3回違反するとアカウントが停止される

#### US-004: 本交際から成婚まで
**As a** 本交際中の会員  
**I want to** プロポーズが成功したら成婚報告を行いたい  
**So that** 正式に退会し、新しい生活を始められる

**受け入れ基準**:
- 本交際ステータスで「成婚退会」ボタンが表示される
- エージェントが承認すると、成婚料の決済リンクが即時発行される
- 決済完了後、アカウントが「成婚退会」ステータスに変更される

### 4.2 エージェント

**ペルソナ**: 山田花子（35歳、非IBJ連盟の個人相談所経営）
会員20名を抱えているが、IBJ連盟の高額な会費がネックで未加入。meetsで低コストに会員データベースを拡充し、成婚数を増やしたい。

#### US-005: エージェント登録とアカウント発行
**As a** 結婚相談所経営者  
**I want to** meetsにエージェント登録し、アカウントを発行してもらいたい  
**So that** 自分の会員をmeetsに登録し、他のエージェントの会員とマッチングさせられる

**受け入れ基準**:
- meets運営に問い合わせ後、HP確認が行われる
- 反社チェックが完了すると、アカウント発行のメールが届く
- ログイン後、成婚料等の料金設定画面が表示される

#### US-006: 担当会員のプロフィール検索とお見合い申し込み
**As a** エージェント  
**I want to** 担当会員の希望条件に合う相手を検索し、お見合いを申し込みたい  
**So that** マッチング機会を増やし、成婚率を高められる

**受け入れ基準**:
- 検索条件（年齢、年収、地域、学歴等）を入力すると、該当する会員一覧が表示される
- 検索条件を5件まで保存できる
- お見合い申し込みボタンを押すと、相手エージェントに通知が送られる
- 相手エージェントが承認すると、日程調整画面に遷移する

#### US-007: CRMでの活動管理
**As a** エージェント  
**I want to** 担当会員の活動状況を一覧で確認したい  
**So that** 適切なタイミングでフォローし、成婚までサポートできる

**受け入れ基準**:
- ダッシュボードに担当会員のステータス（登録/お見合い申込/お見合い実施/仮交際/本交際/成婚）が表示される
- 各会員の最終活動日が表示される
- 会員ごとにメモを記録でき、次回ログイン時に閲覧できる

#### US-008: 成婚報告と決済
**As a** エージェント  
**I want to** 担当会員から成婚報告を受けたら、成婚料を請求したい  
**So that** 自分の報酬を受け取れる

**受け入れ基準**:
- 会員が成婚退会を選択すると、エージェントに承認依頼通知が届く
- 承認ボタンを押すと、会員に成婚料決済リンクが自動送信される
- meets経由決済の場合、10%差し引かれた金額が翌月10日に銀行口座に振り込まれる
- 直接請求を選択した場合、手数料は発生しない

### 4.3 アンバサダー

**ペルソナ**: 鈴木太郎（40歳、国際結婚相談所経営、YouTube登録者5万人）
自分の顧客（主に男性）をmeetsに紹介し、成婚料の一部をキックバックとして受け取りたい。

#### US-009: 紹介コード発行と会員誘導
**As a** アンバサダー  
**I want to** 専用の紹介コードを発行し、自分のチャンネル視聴者に配布したい  
**So that** 紹介した会員が成婚した際に報酬を得られる

**受け入れ基準**:

- 紹介コードはダッシュボードからコピー可能
- 会員登録時に紹介コード入力欄があり、コードを入力すると紐づけが記録される

#### US-010: 成婚料のキックバック受領
**As a** アンバサダー  
**I want to** 自分が紹介した会員が成婚したら、設定された料率でキックバックを受け取りたい  
**So that** 継続的に会員を紹介するモチベーションを維持できる

**受け入れ基準**:
- 紹介会員が成婚すると、アンバサダーダッシュボードに成婚通知が表示される
- 成婚料30万円・料率30%の場合、9万円が報酬として計算される

- 担当エージェントには21万円が配分され、meets側が10%を差し引いた18.9万円が振り込まれる

#### US-011: 紹介実績の可視化
**As a** アンバサダー  
**I want to** 自分が紹介した会員の成婚数と獲得報酬額を確認したい  
**So that** 紹介活動の効果を測定できる

**受け入れ基準**:
- ダッシュボードに紹介会員数、成婚数、獲得報酬額が月次で表示される
- CSVエクスポート機能で詳細データをダウンロードできる

**As a** プラットフォーム管理者  
**I want to** エージェント申請を審査し、承認/却下を判断したい

### 5.1 認証・本人確認

**BR-001: eKYC本人確認**
- AC-001-1: [Given] 会員が運転免許証の画像をアップロードした [When] OCR処理が完了した [Then] 氏名、生年月日、住所がフォームに自動入力される
